# Adding New Articles to RC Journey

This guide covers the complete workflow for publishing a new post — from writing
the markdown to seeing it live on the site.

---

## Directory structure

Each post lives in its own folder under `output/posts/`, named by the URL slug
you want to use. The slug becomes the article's permanent URL
(`https://rcjourney.cloud/<slug>/`), so choose it carefully.

```
output/posts/
└── your-post-slug/
    ├── index.md          ← article content and frontmatter
    └── images/
        ├── cover.webp    ← cover / hero image
        └── *.webp        ← any other images referenced in the body
```

Image files can be JPEG, PNG, or WebP. WebP is preferred — Astro optimizes
them all at build time regardless, but WebP files build faster.

---

## Frontmatter

The schema is strict: a missing or misspelled field fails the build. Copy this
template and fill it in:

```yaml
---
title: "Your Article Title"
author: Brett Buskirk
date: 2026-06-20
categories:
  - "reflection"
tags:
  - "tag-one"
  - "tag-two"
coverImage: "./images/cover.webp"
draft: false
---
```

### Field reference

| Field | Type | Notes |
|---|---|---|
| `title` | string | Required. Quoted to be safe with colons/apostrophes. |
| `author` | string | Required. Displayed as a byline below the article title. e.g. `Brett Buskirk` |
| `date` | YYYY-MM-DD | Required. Controls sort order. |
| `categories` | list | Required. **Exactly one** value from the list below. |
| `tags` | list | Optional. Free-form strings; shown as chips on the article. |
| `coverImage` | path | Required. Must be `./images/<filename>` — relative to the post folder. File must exist or the build fails. |
| `draft` | boolean | Optional, defaults to `false`. Set `true` to exclude from builds. |

### Valid category keys

Pick exactly one. The key maps to the site's editorial section and determines
the section page the article appears under.

| Key | Section label | Section URL |
|---|---|---|
| `shadowed-mirror` | The Shadowed Mirror | `/the-shadowed-mirror/` |
| `reentry-realities` | Reentry Realities | `/reentry-realities/` |
| `reflection` | Reflections | `/reflections/` |
| `the-deep-well` | The Deep Well | `/the-deep-well/` |
| `rcj-info` | About the Journey | `/rcj-info/` |

## Writing the article body

The body is standard Markdown. A few site-specific behaviors to know:

**Cover image deduplication.** If the first image in the body is the same file
as `coverImage`, it's automatically stripped — the hero already renders it at
the top of the page, so it won't appear twice.

**Inline images.** Use standard Markdown image syntax with paths relative to
the post folder:

```markdown
![Alt text describing the image](./images/filename.webp)
```

Astro processes these at build time: it generates responsive WebP variants,
adds correct `width`/`height` to prevent layout shift, and hashes the filenames
for cache-busting.

**Gallery sections.** Add a `## Gallery` heading followed by your images on
consecutive lines. The build pipeline converts this into a responsive image
grid with a lightbox automatically:

```markdown
## Gallery

![Caption for first photo](./images/photo-01.webp)
![Caption for second photo](./images/photo-02.webp)
![Caption for third photo](./images/photo-03.webp)
```

Captions appear below each thumbnail and inside the lightbox. Leave alt text
empty (`![](...) `) if you don't want a caption.

**Links.** Standard Markdown. Internal links should use root-relative paths
(`/another-post-slug/`) so they work in both dev and production.

## Publishing workflow

### 1. Create the post directory and write the article

```bash
mkdir -p output/posts/your-post-slug/images
# copy your images into output/posts/your-post-slug/images/
# create output/posts/your-post-slug/index.md
```

### 2. Test locally

```bash
npm run build        # full build + Pagefind index
npm run preview      # serve at http://localhost:4321
```

Check: article renders correctly, cover image loads, gallery (if any) works,
the post appears on the blog index and its section page.

> `npm run dev` works for writing/editing but **search won't function** in dev
> mode — Pagefind's index is only generated by `npm run build`.

### 3. Commit and push

```bash
git add output/posts/your-post-slug
git commit -m "Add article: Your Article Title"
git push origin main
```

Pushing to `main` triggers an automatic build and deploy on DigitalOcean App
Platform. The live site updates within a minute or two.

## Common mistakes

| Symptom | Cause |
|---|---|
| Build fails with "Expected type" | `categories` key is misspelled or not in the valid list, or a required field like `author` is missing |
| Build fails with "Image not found" | `coverImage` path doesn't match an actual file in `images/` |
| Cover image appears twice | The first body image isn't identical to `coverImage` — they should match exactly (same filename) |
| Gallery shows as links, not images | Used `[alt](path)` instead of `![alt](path)` |
| Post missing from blog/section index | `draft: true` is set, or the `date` is in the future |
| Search doesn't find the new post | Ran `npm run dev` instead of `npm run build` — Pagefind only indexes on full builds |
